// 01 — Coursework

Labs

Five weekly labs, each adding a layer to the Android stack — from first UI to persistent local data.

Lab 01UI

Building the UI

My first Compose layouts taught me how declarative UI replaces XML with composable functions. I learned to think in terms of state and recomposition, and how Modifier chains control sizing, padding, and arrangement.

Lab 02Interaction

Handling Interaction

This lab connected the UI to behaviour — clicks, text input, and remembering state with remember and mutableStateOf. I saw how unidirectional data flow keeps the screen and its state in sync as the user interacts.

Lab 03Material Design

Material Design

I applied Material 3 components, theming, and dynamic color for a consistent, accessible look. Working with typography scales and color roles showed how a real design system speeds up development.

Lab 04Navigation

Navigation

I implemented multi-screen navigation with the Navigation component, passing arguments between destinations and managing the back stack. This became the structural foundation for Project 1's screens.

Lab 05Room

Room Persistence

Room gave my app type-safe local storage. I defined entities, DAOs, and a database, then observed data reactively with Flow so the UI updates automatically whenever records change.

// progression

UI → Interaction → Material → Navigation → Data

Each lab stacked onto the last. By Lab 5 I could combine layout, state, theming, navigation, and persistence into one coherent app — the groundwork for both projects.

// 02 — Capstone

Projects

Two builds on one shared codebase — a working video player, then a connected edition with data, API, sensors, and cloud.

Project 01 · Lab 4 + Working Features

Video Player App

Problem statement

Many lightweight media players bury basic playback controls and lack a tidy library view. This app puts a clean, navigable interface and the essential playback controls front and centre.

Lessons learned

I combined multi-screen navigation with real playback features, wiring media controls into a Compose UI. Managing player state across navigation and lifecycle taught me to handle configuration changes and clean up resources carefully.

Project 02 · Lab 5 + API + Sensors + Cloud

Video Player — Connected Edition

Problem statement

Building on Project 1, this version adds persistence, online content, and device awareness so the library survives restarts and adapts to the user's context.

Lessons learned

Integrating Room, a remote API, sensors, and cloud services in one app showed me how to coordinate asynchronous data sources. I improved my error handling and learned to keep the UI responsive while syncing local and remote state.

// 03 — Looking back

Reflection

Starting from a single static screen in Lab 1, each lab added a layer — interaction, Material theming, navigation, and finally local data. By the projects, I could combine these skills to ship a working video player, then extend it with an API, sensors, and the cloud. The biggest shift was learning to think in state and data flow instead of step-by-step instructions — and to read docs, debug, and structure a codebase on my own.
